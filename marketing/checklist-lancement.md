# Checklist de publication

## Avant de communiquer

- [ ] La release GitHub est publique et non marquée comme brouillon.
- [ ] `latest.json` renvoie un statut HTTP 200.
- [ ] L’installateur correspond à la version annoncée.
- [ ] Le téléchargement depuis bob59.fr fonctionne.
- [ ] La fiche du site affiche la bonne version.
- [ ] Les notes de version décrivent réellement les changements.
- [ ] Les captures correspondent à l’interface actuelle.

## Pour chaque vidéo

- [ ] Accroche compréhensible dans les deux premières secondes.
- [ ] Fonction réelle montrée à l’écran.
- [ ] Sous-titres présents.
- [ ] Aucun fichier personnel ou chemin sensible visible.
- [ ] Adresse bob59.fr affichée à la fin.
- [ ] Musique et images autorisées pour un usage public.

## Après publication

- [ ] Répondre aux premières questions et commentaires.
- [ ] Noter les demandes de fonctions récurrentes.
- [ ] Vérifier les téléchargements de la release GitHub.
- [ ] Transformer les questions fréquentes en prochaines vidéos.
- [ ] Comparer vues, clics et téléchargements après 24 h et 7 jours.
